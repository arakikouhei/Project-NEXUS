# Project NEXUS v2.0 ハードウェア構成案

## 目的

この文書は、将来の球体AIデバイス用ハードウェア構成を整理する。

## 外装

- 球体
- 底面は安定用に平面化
- 手のひらより大きく、ハンドボール大を想定
- メンテナンス用に分解可能な構造が望ましい

## 表示

上面に小型ディスプレイを配置する。

表示候補：

- バッテリー残量
- 起動状態
- 接続状態
- 簡易返答
- 絵文字
- 警告
- 処理中表示

## 入出力

候補：

- USB-C
- SDカードスロット
- 内蔵マイク
- 小型スピーカー
- Wi-Fi
- Bluetooth

## 内部構成

候補：

- 中央処理ユニット
- AI処理用モジュール
- 電源管理
- バッテリー
- 慣性センサー
- 温度センサー
- 冷却機構
- アンテナ
- 表示制御基板

## 冷却

液冷は将来検討。

ただし、初期段階では以下を優先する。

1. 発熱の少ない構成
2. 低消費電力
3. 空冷または自然放熱
4. 安全性
5. メンテナンス性

液冷は複雑なので、最終段階まで保留する。

## 電源

安全上、バッテリー制御は慎重に扱う。

必要な方針：

- 過充電保護
- 過放電保護
- 温度監視
- 異常時停止
- 充電中の安全制御

## v2.0での結論

v2.0ではハードウェア実装ではなく、構成案と安全方針を完成させる。

実物製造はv2.1以降の課題にする。
