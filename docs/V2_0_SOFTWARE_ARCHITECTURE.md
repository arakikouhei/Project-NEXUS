# Project NEXUS v2.0 ソフトウェア構成案

## 目的

この文書は、球体AIデバイスに搭載する将来ソフトウェア構成を整理する。

## 基本方針

Project NEXUSはローカル優先で動作する。

基本は次の順番。

1. ローカル処理
2. ローカル記憶
3. ローカルファイル確認
4. ユーザー確認
5. 必要な場合のみ外部連携

## 主要モジュール候補

- core
- ai
- memory
- device
- network
- security
- voice
- dashboard
- file index
- production support
- logger

## 音声処理

将来の流れ：

1. マイク入力
2. 音声認識
3. テキスト化
4. コマンド判定
5. 安全確認
6. 実行
7. 応答生成
8. 音声出力

## ディスプレイ表示

表示候補：

- 起動中
- 待機中
- 聞き取り中
- 処理中
- 応答中
- エラー
- バッテリー
- 温度
- ネットワーク

## セキュリティ

必要な方針：

- 外部通信は最小限
- 有料APIは明示確認
- ログに重要情報を出しすぎない
- 危険操作は確認制
- 自律削除は禁止
- 自律送信は禁止

## v2.0での結論

v2.0では、球体AI化に向けたソフトウェア構成案を完成させる。

実装はv2.1以降で段階的に行う。
